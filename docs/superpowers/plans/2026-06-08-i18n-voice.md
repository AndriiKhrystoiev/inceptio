# VOICE phase — Localize server-composed astrology copy — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** Compose the astrology VOICE copy in the request locale (de/fr/es-419/pt-BR), thread `X-Locale` through composition + both cache keys, and fill the client `voice.*` files — server-only follow-up to the CHROME phase.

**Architecture:** Leaf `Record<Locale,string>` (a `Localized<string>`) resolved once at each terminal composer function via a tolerant `localize(value, locale)` helper that accepts a plain string (= English-everywhere) OR a Record. This decouples the **locale-threading spine** (Task 0) from the per-dictionary translation (parallel) — the spine wraps every leaf-read in `localize()`, so dictionaries migrate string→Record independently with no coordination.

**Tech Stack:** Cloudflare Worker (Vitest, TS strict), i18next on mobile (already wired). Spec: `docs/superpowers/specs/2026-06-08-i18n-voice-design.md`. Archaeology: the Phase-1A composer audit (counts/call-path/hazards).

## Conventions
- **Locale = `'en'|'de'|'fr'|'es-419'|'pt-BR'`.** `localize(v, locale)` = `typeof v === 'string' ? v : (v[locale] ?? v.en)`.
- Worker tests: `cd workers/api-proxy && npx vitest run`. Mobile: `cd apps/mobile && npx vitest run`.
- **Glossary-guided:** every dictionary translation consults `docs/superpowers/glossary/i18n-termbase.md` for the locale's traditional astrology renderings (pt-BR *Lua Fora de Curso*, *Combustão*, etc.). Drafts pending best-effort community review; en authoritative.

- [ ] **Step 2 — `resolveLocale`.** In `lib/locale.ts` add `export function resolveLocale(raw: string|null): Locale { return (raw && (['de','fr','es-419','pt-BR'] as const).includes(raw as any)) ? raw as Locale : 'en'; }` (absent/unknown → 'en'). Unit-test it.

## Task T — Guard flip + char-budget split + deferred set (after all D)

**Files:** `workers/api-proxy/src/translations/__tests__/lint-library.test.ts` (split), a new server `voice-coverage.test.ts`, and `apps/mobile/src/i18n/__tests__/coverage.test.ts` (flip).

- [ ] **Step 1 — Server all-locales leaf coverage.** Test asserting every localized leaf `Record` in every dictionary has all 5 locale keys (no missing locale). The deferred named set `VOICE_DEFERRED = ['status-lines','empty-state']` is exempt — assert the set is EXACTLY those two (can't grow), tied to the "translate when saved-search fan-out ships" comment.
- [ ] **Step 2 — lint split.** In `lint.ts`/its test: **language-content checks (forbidden-word/horizon/`.includes()`/3-day rule) run on en ONLY**; **structural + char-budget checks run on ALL 5 locales.** Do not scope the budget gate to en.
- [ ] **Step 3 — Card-bound vs screen-bound budgets.**
  - **Card-bound (hard) — gate the COMPOSED synthesizer OUTPUT, not individual leaves.** The card renders the *synthesized* headline = either a `HEADLINES` custom entry OR `GENERIC_HEADLINE_STEMS[activity]` template applied to a specific `{lead}` (a factor `phrase_short`). Budgeting the template and the lead separately misses the long-template + long-lead combination that still overflows. **Enumerate the synthesizer's possible outputs per locale** — for each activity: every `HEADLINES` entry; every `NO_VIABLE_HEADLINES` entry; and every `GENERIC_HEADLINE_STEMS` template × every possible lead (the set of factor `phrase_short` values that can reach the generic path for that activity/status) — and assert **each composed output ≤ the card capacity** for all 5 locales. (Worst-case shortcut acceptable: `max(template-with-lead-substituted)` over the cross-product; the bound is longest-template ⊕ longest-eligible-lead.) German overflow here **FAILS the build** (card correctness).
  - **Derive the card capacity EMPIRICALLY, not a guessed number.** Compute from the real render — headline font (26px display-italic @ 9:16 / 19px @ 1:1) × 360px content width × the allowed line count — and **verify it against an actual `MomentCard` render** (render the longest English composed headline + a synthetic at-capacity German string; confirm no clip/overflow on both card ratios). Pin the measured char/width ceiling. A guessed number either lets German overflow or over-truncates. *(This also closes the latent EN gap: the card headline had no budget at all before.)*
  - **No runtime backstop** (numberOfLines/adjustsFontSizeToFit on MomentCard.js:36) — the composed-output gate is comprehensive because synthesizer outputs are enumerable, and a backstop would pull client work into this server phase. Skip it.
  - **Screen-bound (all-locales, soft):** daily-note headline ≤48 / supporting ≤140 / severity ≤150 — assert for all 5 locales; a failure is the German-layout-pass signal, but keep it a real assertion so overflow is detected (do NOT scope the budget gate to en).
- [ ] **Step 4 — Client coverage flip.** In mobile `coverage.test.ts`: `voice.*` keys must now exist in ALL 5 locales (remove the en-only inverse); `onboarding:subhead` stays the length-1 CHROME exception (unchanged).
- [ ] **Step 5 — Verify + commit** both packages green; `test(i18n): VOICE coverage flip + card/screen budget split + deferred named set`.
