# Splash Screen Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A static branded splash — Inceptio symbol + "Inceptio" wordmark centered on `#0F0A1F`, shown while the app boots, with no white flash.

**Architecture:** A dev-time Node script composes `assets/splash-icon.png` (transparent) from the existing transparent symbol (`icon-android-foreground.png`, auto-trimmed) + the wordmark rendered in the bundled Fraunces font via `@resvg/resvg-js`. The native splash is configured through the `expo-splash-screen` config plugin in `app.json`. A one-line `App.js` safety fix guarantees the splash is never frozen by a boot error. Timing (`preventAutoHideAsync`/`hideAsync`) already exists and is untouched.

**Tech Stack:** Expo SDK 55 (CNG/prebuild workflow), `expo-splash-screen` config plugin, `@resvg/resvg-js` (dev-time PNG rasterizer), `pngjs` (dev-time PNG read/crop), Fraunces TTF from `@expo-google-fonts/fraunces`, Vitest (node env).

## Global Constraints

- Background color is `#0F0A1F` (bg-deep) everywhere; wordmark fill is `#F5EFE4` (cream). App is dark-only.
- The splash artwork PNG has a **transparent** background (the native splash fills `#0F0A1F` and centers the image).
- `@resvg/resvg-js` pinned to **`2.6.2`** (stable; never the `next`/alpha tag). It is a **devDependency** — never imported by app/runtime code (never enters the RN/Hermes bundle).
- resvg font gotcha: the SVG `font-family` and `defaultFontFamily` must equal the .ttf's **internal** family name. For `Fraunces_500Medium_Italic.ttf` that name is **`"Fraunces Medium"`** (verified via the name table). Using the filename or a wrong family makes glyphs silently drop.
- `icon-android-foreground.png` is **read-only** input (it is also `android.adaptiveIcon.foregroundImage`). The script writes ONLY to `assets/splash-icon.png`.
- Auto-trim is mandatory: the foreground has asymmetric ~20%/side transparent padding (content bbox `x[211–826] y[173–801]`); the script must crop to the alpha bbox (re-centering) before placing.
- Build script is `.mjs` (the package is CommonJS; `.mjs` gives ESM without flipping `type`).
- Tests live under `apps/mobile/src/lib/__tests__/` (the only globs Vitest includes — see `vitest.config.ts`). Do NOT put tests in `scripts/__tests__/` (silently skipped).
- Android 12+ shows the symbol only (system circular mask) — accepted, documented; not a bug.
- All commands run from `apps/mobile/` (it is NOT a root npm workspace member; `npm -w` forms fail).

- [ ] **Step 4: Write the composer script**

Create `apps/mobile/scripts/build-splash.mjs`:
```js
import { readFileSync, writeFileSync } from 'node:fs';
import { fileURLToPath } from 'node:url';
import { dirname, join } from 'node:path';
import { PNG } from 'pngjs';
import { Resvg } from '@resvg/resvg-js';

const ROOT = join(dirname(fileURLToPath(import.meta.url)), '..'); // apps/mobile
const SYMBOL_SRC = join(ROOT, 'assets/icon-android-foreground.png'); // read-only input
const FONT_PATH = join(ROOT, 'node_modules/@expo-google-fonts/fraunces/Fraunces_500Medium_Italic.ttf');
const OUT = join(ROOT, 'assets/splash-icon.png');

// ── Tunable layout constants (px) ───────────────────────────────────────────
const CANVAS = 1242;                  // square output
const SYMBOL_W = 460;                 // on-canvas symbol width
const GAP = 64;                       // symbol → wordmark gap
const WORDMARK = 'Inceptio';
const FONT_FAMILY = 'Fraunces Medium'; // .ttf internal family name (name-table id 1)
const FONT_PX = 150;
const TEXT_FILL = '#F5EFE4';
const ALPHA_THRESHOLD = 8;

// 1. Read symbol + compute tight alpha bbox (mandatory auto-trim/re-center)
const png = PNG.sync.read(readFileSync(SYMBOL_SRC));
let minX = png.width, minY = png.height, maxX = -1, maxY = -1;
for (let y = 0; y < png.height; y++) {
  for (let x = 0; x < png.width; x++) {
    const a = png.data[(png.width * y + x) * 4 + 3];
    if (a > ALPHA_THRESHOLD) {
      if (x < minX) minX = x;
      if (x > maxX) maxX = x;
      if (y < minY) minY = y;
      if (y > maxY) maxY = y;
    }
  }
}
const bw = maxX - minX + 1;
const bh = maxY - minY + 1;

// 2. Crop to bbox
const cropped = new PNG({ width: bw, height: bh });
for (let y = 0; y < bh; y++) {
  for (let x = 0; x < bw; x++) {
    const si = (png.width * (minY + y) + (minX + x)) * 4;
    const di = (bw * y + x) * 4;
    cropped.data[di] = png.data[si];
    cropped.data[di + 1] = png.data[si + 1];
    cropped.data[di + 2] = png.data[si + 2];
    cropped.data[di + 3] = png.data[si + 3];
  }
}
const symbolB64 = PNG.sync.write(cropped).toString('base64');

// 3. Geometry — symbol above wordmark, whole block vertically centered
const symbolH = Math.round((SYMBOL_W * bh) / bw);
const blockH = symbolH + GAP + FONT_PX;
const blockTop = Math.round((CANVAS - blockH) / 2);
const symbolX = Math.round((CANVAS - SYMBOL_W) / 2);
const symbolY = blockTop;
const textBaselineY = blockTop + symbolH + GAP + Math.round(FONT_PX * 0.72);

// 4. Build SVG (transparent background)
const svg = `<svg xmlns="http://www.w3.org/2000/svg" width="${CANVAS}" height="${CANVAS}" viewBox="0 0 ${CANVAS} ${CANVAS}">
  <image x="${symbolX}" y="${symbolY}" width="${SYMBOL_W}" height="${symbolH}" href="data:image/png;base64,${symbolB64}"/>
  <text x="${CANVAS / 2}" y="${textBaselineY}" text-anchor="middle" font-family="${FONT_FAMILY}" font-style="italic" font-size="${FONT_PX}" fill="${TEXT_FILL}">${WORDMARK}</text>
</svg>`;

// 5. Rasterize with the bundled Fraunces font (no background = transparent)
const resvg = new Resvg(svg, {
  fitTo: { mode: 'width', value: CANVAS },
  font: { fontFiles: [FONT_PATH], loadSystemFonts: false, defaultFontFamily: FONT_FAMILY },
});
writeFileSync(OUT, resvg.render().asPng());
console.log(`wrote ${OUT} — ${CANVAS}x${CANVAS}; symbol bbox ${bw}x${bh} @ (${minX},${minY})`);
```

## Task 3: Anti-freeze safety fix (App.js)

**Files:**
- Modify: `apps/mobile/App.js` (the hydrate chain ~:133-165 and `onLayoutRoot` ~:181)

**Interfaces:**
- Consumes: nothing new. Hardens the existing splash-release gate so a boot error can't freeze the (now branded) splash.

**No unit test:** `App.js` is a JSX component importing React Native / native modules, which Vitest excludes (see `vitest.config.ts` globs — `App.js` is not covered, and RN can't be imported in the node test env). This is a defensive one-liner verified by inspection + the Task 4 visual run.

- [ ] **Step 1: Add `.catch` to the hydrate chain**

In `apps/mobile/App.js`, find the hydrate effect that ends with `setStorageReady(true);` inside `hydrateStorage().then(() => { … });` (around line 133-165). Change the closing of that `.then(...)` from:
```js
      setStorageReady(true);
    });
  }, []);
```
to:
```js
      setStorageReady(true);
    }).catch(() => {
      // Never let a hydrate/init error freeze the (now branded) splash forever.
      // Lift the gate so the app boots; worst case is defaults instead of
      // persisted prefs. See splash spec §4.3.
      setStorageReady(true);
    });
  }, []);
```

- [ ] **Step 2: Guard the hideAsync call**

In the same file, change `onLayoutRoot` (around line 180-182) from:
```js
  const onLayoutRoot = useCallback(async () => {
    if (fontsLoaded && storageReady) await SplashScreen.hideAsync();
  }, [fontsLoaded, storageReady]);
```
to:
```js
  const onLayoutRoot = useCallback(async () => {
    if (fontsLoaded && storageReady) await SplashScreen.hideAsync().catch(() => {});
  }, [fontsLoaded, storageReady]);
```
